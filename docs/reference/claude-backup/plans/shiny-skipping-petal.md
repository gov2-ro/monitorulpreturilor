# Plan: Gas Station Map + Full UAT Coverage + Gas in CI

## Context

Gas pipeline exists (`fetch_gas_prices.py`, `fetch_gas_reference.py`) but is disconnected from CI and limited to ~20 UATs in the `uats` table. The fuel page and trends page already have structure for gas data, but they run off static/manual data. Users want:
1. A Leaflet map showing gas stations with latest prices per station in the popup
2. Full national UAT coverage (not just the initial ~20)
3. Gas fetching automated in CI so trends update daily

Price variation observation: GPL prices vary meaningfully across networks/stations; other fuels have less spread but station-level variation still exists — worth fetching all UATs.

Smart clustering (used for retail) does **not** apply to gas: the gas API is per-UAT (`GetGasItemsByUat`), not per-lat/lon. Each UAT is already a geographic grouping. No clustering needed.

### 2. `generate_site.py` — add `gen_gas_map()` and `load_gas_map_data()`

**New function `load_gas_map_data(conn)`** — query latest price per station per fuel type:

```sql
SELECT gs.id, gs.name, gs.addr, gs.lat, gs.lon,
       gn.name AS network, gn.id AS net_id,
       gpr.name AS fuel, gp.price, gp.price_date
FROM gas_stations gs
JOIN gas_networks gn ON gs.network_id = gn.id
JOIN gas_prices gp ON gp.station_id = gs.id
JOIN gas_products gpr ON gp.product_id = gpr.id
WHERE (gp.product_id, gp.station_id, gp.price_date) IN (
    SELECT product_id, station_id, MAX(price_date)
    FROM gas_prices GROUP BY product_id, station_id
)
AND gs.lat IS NOT NULL AND gs.lon IS NOT NULL
```

Pivot in Python into per-station dicts:
```python
{id: {name, addr, lat, lon, network, net_id, prices: {fuel_name: price, ...}, price_date}}
```

**New function `gen_gas_map(conn, out_path)`** — modelled after `gen_stores_map()`:
- Embed `const stations = [...]` as JSON (id, name, addr, lat, lon, net, c, prices dict, price_date)
- Network colors: reuse existing `net_color()` (already covers Rompetrol, LUKOIL, MOL, OMV, PETROM, SOCAR, Gazprom)
- SVG circle markers (same 14×14 px as stores map), colored by network
- **Popup**: station name (bold), network (colored), address, small price table (fuel → RON/L), date
- Legend: checkbox per network + count
- `markerClusterGroup` with `maxClusterRadius: 40`
- Page title: "Hartă Stații Carburanți"

Wire into `main()`: call `gen_gas_map(conn, out / "gas_map.html")`.

**File modified**: `generate_site.py`

### 4. `.github/workflows/ci_prices.yml` — add gas steps

Add after the retail fetch step, before `generate_site.py`:

```yaml
# Weekly gas reference refresh (Mondays)
- name: Refresh gas reference data
  if: github.event_name == 'schedule' && startsWith(github.event.schedule, '0 4 * * 1')  
  run: |
    source venv/bin/activate
    python fetch_gas_reference.py data/prices_ci.db

# Daily gas price fetch
- name: Fetch gas prices
  run: |
    source venv/bin/activate
    python fetch_gas_prices.py data/prices_ci.db --max-runtime 900
```

`--max-runtime 900` (15 min) keeps total CI time within the 120-min GitHub Actions limit.
The gas checkpoint is stored in `data/gas_checkpoint.json`; add to the commit step.

**File modified**: `.github/workflows/ci_prices.yml`
